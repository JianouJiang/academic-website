Generalization & Physics Injection

Candidate

与此同时还有一个痛点：我们很多模型只能在 training range 里 behave and perform。如果给一个没见过的 sample，它是否还能 behave？

经典物理定律适配范围非常广。比如 F = ma，你可以 plug any numbers in it，F 都可以给你一个结果。可是基于神经网络的模型，解释范围更小。比如用 neural net 做一个 polynomial fit，你可以解释它。但当 a 超过训练范围（比如训练时喂的数据是 10 到 100），如果测试 a = 110，will it perform？这是一个很大的痛点。

我相信如果把一些物理注入进去，无论是注入到神经网络里面，还是注入到它的 loss function 里，可以帮助它的 generalization 更好。

一个是把黑变得更明，一个是把可解释性变得更广。

除此以外它的一个优势是可以作为简化算法——算得更快，也可以算得还不错的准。前期训练成本另算，但 the story is: if we train the model beforehand, 我们只需要训练一次，之后它就可以 package，卖给其他人，他们不需要重复训练，或者只需要在大模型上进行微调，训练成本就可以被忽略不计。

Methodology: Physics Loss

Candidate

两个模型：一个是利用 physics 注入到神经网络内部，一个是用 physics 注入到 loss function 里面。

现在 loss function 都是 data loss。一个比较新的领域就是 physics loss——你需要看模型有多符合物理定律。如果不符合 conservation of energy、conservation of momentum，如果有一些 residual of the governing equations，那它的 punishment 就很高。

所以这也是一种训练方式，in addition to data loss。

Q&A: Optimization Method

Interviewer A

你使用了什么样的方法去解决这个问题？我理解这个问题是多变量输入、多变量输出。我们面对的很多工厂场景也是这样的问题。你使用了什么方法去模拟它？或者说神经网络在中间起了什么样的作用？

Candidate

其实也不用那么复杂。神经网络只是一个 multi-dimensional grid search。很多变量，比如变量 A 从 1 到 10，变量 B 从 100 到 1000，我们可以暴力搜。但当 dimension 很高怎么办？就涉及到前期的取舍。

首先排除哪些变量不是重要变量，然后看哪些变量 sensitivity 很高。如果 sensitivity 很高，就在 grid search 中增加精度——比如从 1 到 10 不能只搜 10 个，而是搜 100 次，增加精度。然后根据文献或找 experts 交流、基于过去经验去进一步缩小范围。

以上本身不会涉及太多神经网络。如果完全自动化也可以，但在我两三年前做的时候没有用到。有时候我们还要考虑现实因素。比如 PC board 我想设计成 15 × 25，但不行，因为要 ship to abroad——集装箱有尺寸限制，如何装满集装箱取决于每个 PC board 的尺寸，而不仅仅是散热。

所以最终得往上汇报：到底是散热重要，还是装满集装箱 ship to abroad 重要。

PhD Thesis: Wall Function

Candidate

好。我不知道你们对背景了不了解，我简要介绍一下。

Interviewer A

相关材料可以投到屏幕上吗？

Candidate

可能现在还不太方便，等文章发出去以后可以分享。

我们说的有限元去解偏微分方程，一个很直观的点是：它越精确，我们需要越多的 grid points 或 finite elements、finite volume。就跟看电视一样——越精细清晰，像素就越多。但像素越多就代表计算更 expensive。

所以 all we want 就是找到简化算法。在流体和固体 interact 的表层，现象是最有意思的——所有的 chemical reaction、thermal exchange、fluid exchange 都发生在这个薄表面。固态和液态、液态和气态之间的交互。这么一小层区域里需要非常多的 finite element、非常多的像素才能精确表达里面的 dynamics。但话说回来，太贵了。

所以我们能不能在一个还不错的高度选一个 point，下面就没有点了、没有像素了？在这个点，我们用经验公式表达下面的交互——选一个点，根据它来表达下面的 velocity profile 或 thermal profile。这就是壁面函数 (wall function)。

但壁面函数有很多假设。比如它 rely on boundary layer equilibrium，需要底下的 boundary layer 是 well-developed。但当特定的情况发生，比如有 adverse pressure gradient、有 separation flow 的时候，假设就 broken 了，经验公式就不再成立了。

我的博士提出的新算法就是用基于机器学习的 algorithm 去 develop this universal wall function。什么叫 universal？就是无论什么情况、什么 geometry，any Reynolds number you can think of, any geometry, any flow types, whether you have adverse pressure gradient or not, whether you have coupled temperature field or chemical reactions，它都可以解释。

这就涉及到投入大量训练数据。你不能只给它某一种 flow field 的数据，because then it doesn't generalize。你需要做非常精确的特征编码 (feature encoding)。

Feature Variables

Candidate

Feature variables 是什么？就是很多 non-dimensionalized combinations of physical variables。比如我们经常涉及到的 y⁺ 或 u⁺，或者 velocity scaled by friction velocity——它是不同物理量的组合。

就像我们做量化项目的时候，基于因子来判断：把不同 variables 组合成一个因子，然后预测未来因子。这个就是物理学里的因子。我叫它 feature variables（特征物理量）。

特征物理量可以放在三个地方：可以作为 input layers，可以作为 hidden layers，也可以作为 output layers。

我两篇博士课题的 paper 分别 investigate 把 feature variables 放在 hidden layers 的 neurons 里面，以及把 feature variables 放在 output layers 作为 loss functions，它对整个 model 的帮助是多少。

Paper 1: Physics-Constrained Loss Function

Interviewer A

第一条是：开发了一种物理约束的损失函数的前馈神经网络，用于拟合分离边界层中的近壁速度分布，通过二维扩散器和高保真模拟验证，结果显示较高的一致性和可扩展性。

Candidate

对，就是我刚才所说的——把 feature variables 放在 output layers、input layers 或 hidden layers，它对整个 model performance 是多少。

我们有几个 metrics。一个是 accuracy：在 normal attached flow condition 下和 conventional wall function 相比做得很不错。在 flow separation 发生的情况下又怎么样？因为 conventional wall function cannot perform in flow separation——如果 our wall function can perform during flow separation, that's a big breakthrough。

Q&A: How Feature Variables Enter Hidden Layers

Interviewer A

你把特征因子放到了 hidden layer 或 output layer。这个东西怎么放进去的？因为我感觉 feature 应该是靠数据自己学出来的，尤其 hidden layer。这是加了一个类似 ResNet 的通道吗？还是什么方式？

Candidate

其实没有那么复杂。一开始纯用 neural net——MLP-based，only one single hidden layer for the velocity profile section。

Interviewer A

Candidate

对——不是说一定是这样的，但 for the sake of transparency：层数多了物理量就解释不清楚了，只有一层的情况下物理量才能解释清楚。

Interviewer A

你有多大规模参数的 MLP？

Candidate

不一定。比如在最新的论文里，有 4、3、8 和 16 neurons in the hidden layer。neurons 个数其实对应的是特定物理场景下流的分层。

我也做过更多层和更多 neurons 的 hyperparameter study。但最后发现层数多了没有太大帮助，反而会让模型 overfit。

Q&A: Correlation with Physics

Interviewer A

Candidate

Interviewer A

Q&A: Dimensionless Numbers & Feature Variables

Interviewer B

我想问的是，分析的是瞬态还是稳态？包括如果有强压力梯度或分离流，传统的 y⁺ 或 u⁺ 两个无量纲数如果不足以描述这种流动状态，在你的学习模型里面，是继续在现有框架上打补丁，还是引入新的无量纲数群和物理特征？

Candidate

两个大问题，一个一个回答。

第一：瞬态还是稳态？其实都 OK，取决于我们用 RANS、LES 还是其他算法，取决于 geometry。

比如一个 diffuser——就是一个渐扩的通道，流会在底下形成 vortex，majority of the flow would come out of this area。这是一个比较低 slope 的 diffuser。

如果是 backward-facing step（一个台阶），那 vortex 就会非常多，要 reach 所谓的 steady-state 是不太可能的——你只能取得一个 statistical stability（统计学意义上的 stability）。最终取决于我们有多长时间、花多少数据能解决问题。Both works。

第二：无量纲数够不够？一开始我们会有无限多的无量纲数——所有我们能想到的全都放进去了。y⁺、u⁺、雷诺数，这些都是通过大量文献阅读生成的。ChatGPT、Claude 最擅长这些事——生成很多无量纲数的组合。

但你把它们喂进去做进一步分析后——for example, good old PCA——你会发现很多 feature variables 是一个东西。比如 y⁺ 和 y⁺ 除以某个 viscous length scale，它只是前面加一个系数，在我们的训练条件下那个系数不变，那不就是同一个 feature variable 嘛。

所以回答您的问题：我们不是从少的无量纲数池子里去解释模型、不行再加。而是一开始就有一堆——all possible feature variables，所有无量纲数都在池子里，然后逐渐减少。这是和您说的正好相反。
